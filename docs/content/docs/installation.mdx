# Installation & Setup

Get started with generate-metadata in minutes. This guide covers installation, basic configuration, and your first metadata generation.

## Installation

Install the generate-metadata package using your preferred package manager:

<Tabs items={["npm", "pnpm", "yarn", "bun"]}>
  <Tab value="npm">```bash npm install generate-metadata ```</Tab>
  <Tab value="pnpm">```bash pnpm add generate-metadata ```</Tab>
  <Tab value="yarn">```bash yarn add generate-metadata ```</Tab>
  <Tab value="bun">```bash bun add generate-metadata ```</Tab>
</Tabs>

## Framework Support

Generate-metadata provides framework-specific imports for optimal integration:

- **Next.js**: `generate-metadata/next`
- **TanStack Start**: `generate-metadata/tanstack-start`

## Environment Setup

### 1. Get Your DSN

Sign up at [generate-metadata.com](https://generate-metadata.com) to get your DSN.

### 2. Configure Environment Variables

Add your DSN to your environment variables:

<Tabs items={["Next.js", "TanStack Start", "General"]}>
  <Tab value="Next.js">
    ```bash title=".env.local" NEXT_PUBLIC_GENERATE_METADATA_DSN=your_dsn_here
    ```
  </Tab>
  <Tab value="TanStack Start">
    ```bash title=".env" GENERATE_METADATA_DSN=your_dsn_here ```
  </Tab>
  <Tab value="General">
    ```bash title=".env" GENERATE_METADATA_DSN=your_dsn_here ```
  </Tab>
</Tabs>

## Development Mode

For development, you can pass `undefined` as the DSN to disable API calls and use only fallback metadata:

```ts
const client = new GenerateMetadataClient({
  // Skip API calls in development
  dsn: undefined,
});
```

Useful for local development, CI/CD, and testing.

## Basic Usage

Here's a minimal example to get you started:

<Tabs items={['Next.js', 'TanStack Start']}>
  <Tab value="Next.js">
    ```ts title="lib/metadata.ts"
    import { GenerateMetadataClient } from "generate-metadata/next";

    export const metadataClient = new GenerateMetadataClient({
      dsn: process.env.NEXT_PUBLIC_GENERATE_METADATA_DSN
    });
    ```

    ```tsx title="app/page.tsx"
    import { metadataClient } from "@/lib/metadata";

    export const generateMetadata = metadataClient.getMetadata(() => ({
      path: "/",
      fallback: {
        title: "My App",
        description: "Welcome to my application"
      }
    }));

    export default function HomePage() {
      return <div>Welcome to my app!</div>;
    }
    ```

  </Tab>
  <Tab value="TanStack Start">
    ```ts title="lib/metadata.ts"
    import { GenerateMetadataClient } from "generate-metadata/tanstack-start";

    export const metadataClient = new GenerateMetadataClient({
      dsn: process.env.GENERATE_METADATA_DSN
    });
    ```

    ```tsx title="routes/index.tsx"
    import { metadataClient } from "@/lib/metadata";

    export const head = metadataClient.getHead(() => ({
      path: "/",
      fallback: {
        meta: [
          { name: "title", content: "My App" },
          { name: "description", content: "Welcome to my application" }
        ]
      }
    }));

    export default function HomePage() {
      return <div>Welcome to my app!</div>;
    }
    ```

  </Tab>
</Tabs>

## Verification

To verify your setup is working correctly:

1. **Check API Connection**: Look for any console warnings about failed API calls
2. **Inspect Metadata**: Use your browser's developer tools to inspect the generated `<meta>` tags
3. **Test Fallbacks**: Try setting `dsn: undefined` to ensure fallback metadata works

## Common Issues

### TypeScript Errors

If you encounter TypeScript errors, ensure you have the latest version installed and your `tsconfig.json` includes:

```json title="tsconfig.json"
{
  "compilerOptions": {
    "moduleResolution": "bundler",
    "allowSyntheticDefaultImports": true,
    "esModuleInterop": true
  }
}
```

### Environment Variables Not Loading

- Next.js: Use `NEXT_PUBLIC_` prefix
- Check `.env` file location and git ignore

### API Connection Issues

- Verify DSN and network connectivity
- Check console for error messages
